Punctuation

Puntuation is represeneted in the inscr_graphs table by the punc field, and in the inscr_graph.class.php class by the public member $punc.

Punctuation represented in this way is editorial punctuation. It does not refer to marks or signs appearing in the original inscription. (They would be encoded as graphs inline with the text, not as markup.)

Punctuation associated with a particular graph is represented using a single integer, the individual bits of which indicate the presence or absence of the possible punctuation characters. Punctuation comes in two flavors: pre-punctuation is punctuation that is inserted before the graph it is associated with in the stream of text. Post-punctuation refers to punctuation that is inserted after the graph with which it is associated. The list below shows the punctuation currently permitted by ECDB, with their corresponding bits. Currently, a two-byte integer is wide enough to accommodate all combinations of permitted punctuation, with one bit to spare.

  • LEFTQUOTE = 1 = 2^0 “
  • LEFTINNERQUOTE = 2 = 2^1 ‘
  • LEFTTITLE = 4 = 2^2 《
  • RIGHTTITLE = 8 = 2^3 》
  • PERIOD = 16 = 2^4 。
  • QUESTION = 32 = 2^5 ？
  • EXCLAMATION = 64 = 2^6 ！
  • LISTCOMMA = 128 = 2^7 、
  • COMMA = 256 = 2^8 ，
  • COLON = 512 = 2^9 ：
  • RIGHTINNNERQUOTE = 1024 = 2^10 ’
  • RIGHTQUOTE = 2048 = 2^11 ”
  • TAB = 4096 = 2^12 (indent, for verse sections, etc.)
  • NEWLINE = 8192 = 2^13 (start new line, for verse sections, paragraphs in long prose passages or speeches, etc.)
  • SEMICOLON = 16384 = 2^14 ;
  • PREPUNC = [NEWLINE, TAB, LEFTQUOTE, LEFTINNERQUOTE, LEFTTITLE]
  • POSTPUNC = [RIGHTTITLE, PERIOD, QUESTION, EXCLAMATION, LISTCOMMA, COMMA, COLON, SEMICOLON, RIGHTINNERQUOTE, RIGHTQUOTE]

When displaying text, the order of punctuation characters should follow their order within the PREPUNC and POSTPUNC lists. I.e. an EXCLAMATION always displays before a RIGHTINNERQUOTE, and a PERIOD always comes after a RIGHTTITLE but before a RIGHTQUOTE.

The items in the following list are those which reliably indicate syntactic sentence divisions. This is useful for dividing a source text into sentence when importing to ECDB. Note that in the practice of many editors, COMMA often coincides with syntactic sentence divisions, but not reliably. This needs to be disambiguated subsequently.

SENTENCEPUNC = [PERIOD, QUESTION, EXCLAMATION, COLON, SEMICOLON]

Under this arrangment, quotes can be nested, but only to a depth of two.
